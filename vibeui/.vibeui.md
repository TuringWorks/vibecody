# VibeUI Workspace Rules

## Icon System

This project uses a **custom thin SVG icon system** — do NOT use lucide-react.

### How to use icons

```tsx
import { Icon } from "./components/Icon";
// or with a relative path:
import { Icon } from "../../components/Icon";

// Usage — size is optional (defaults to 16)
<Icon name="settings" size={20} />
<Icon name="file-code" size={14} />
```

### Icon theming

Icons inherit `color` from their parent via `stroke="currentColor"`.
Stroke weight is controlled globally via the CSS variable `--icon-stroke` (default: `1.25`).

```css
/* Override stroke per context: */
.my-component { --icon-stroke: 1.5; }
```

### Rules

- Always use `<Icon name="..." />` — never `<SomeIcon size={x} strokeWidth={y} />` from lucide-react.
- If the icon you need is not in the list above, pick the closest match or add it to `src/components/Icon.tsx`.
- Icon size follows context: `14` for inline/labels, `16` for buttons/lists, `20` for activity bar.
- Do not hardcode `color` on icons — let CSS `currentColor` handle theming.

## Code Style

- Prefer editing existing files over creating new ones.
- Use CSS variables from `App.css` (`--bg-primary`, `--text-secondary`, `--accent-blue`, etc.) — no hardcoded hex colors.
- Use `var(--radius-sm/md/lg)` for border-radius, `var(--transition-fast/smooth)` for transitions.
- Components live in `src/components/`. Panels live in `src/components/panels/`.
