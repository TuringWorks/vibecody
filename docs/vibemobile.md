VibeMobile is a Flutter companion app for remote management of VibeCody CLI and desktop sessions.

## Platforms

| Platform | Status | Notes |
|----------|--------|-------|
| iOS | Supported | Requires Xcode, iOS 14+ |
| Android | Supported | API 21+ (Android 5.0) |
| macOS | Supported | Native desktop app |
| Linux | Supported | GTK-based |
| Windows | Supported | Win32 app |
| Web | Supported | Any modern browser |

## Features

### QR Code Pairing
Scan a QR code generated by `vibecli serve` + `/pair` to instantly connect your mobile device to a running VibeCody instance. No manual configuration needed.

### Remote Chat
Send messages to any AI provider (Claude, GPT, Gemini, Ollama, etc.) through your connected VibeCody instance. Full streaming support with markdown rendering.

### Machine Management
Register and monitor multiple VibeCody machines:
- Health status with heartbeat monitoring
- CPU/memory metrics
- Active provider and model info
- Session count and task queue

### Session Browser
View and manage active agent sessions:
- Session list with status indicators
- Resume or cancel running sessions
- View session history and traces

### Push Notifications
Get notified when:
- Agent tasks complete
- Security alerts fire
- Machine goes offline/online

## Architecture

VibeMobile uses Provider for state management and communicates with VibeCody's HTTP serve API.

```
VibeMobile (Flutter)
    ↓ HTTP/REST
VibeCLI serve (Rust/Axum)
    ↓
AI Providers / Agent Loop / Tools
```

### State Management
- `AuthService` — Device pairing, token management, machine registry
- `ApiClient` — HTTP wrapper for VibeCody serve API
- `NotificationService` — Push notification handling

### Secure Storage
API tokens and pairing secrets are stored in platform-native secure storage:
- iOS: Keychain Services
- Android: EncryptedSharedPreferences (Keystore-backed)
- macOS/Linux/Windows: OS credential store via `flutter_secure_storage`

## Getting Started

### Prerequisites
- Flutter SDK ≥3.2.0
- Platform-specific tools (Xcode for iOS/macOS, Android Studio for Android)

### Quick Start

```bash
cd vibemobile
flutter pub get
flutter run
```

### Pairing with VibeCLI

1. Start VibeCLI server: `vibecli --serve --port 7879`
2. In the REPL, run `/pair` to generate a QR code
3. Open VibeMobile and tap **Pair**
4. Scan the QR code — done!

### Manual Connection

1. Start VibeCLI server: `vibecli --serve --port 7879`
2. Open VibeMobile → **Connect Manually**
3. Enter `host:port` and API token

## API Endpoints Used

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/health` | GET | Server health check |
| `/status` | GET | Provider/model info |
| `/chat` | POST | Send chat message |
| `/sessions` | GET | List agent sessions |
| `/pair` | POST | Register device pairing |
| `/dispatch/register` | POST | Register machine |
| `/dispatch/heartbeat` | POST | Machine heartbeat |

See [VibeCLI Server Mode](/vibecli/#server-mode) for complete API documentation.
