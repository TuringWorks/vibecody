# Kotlin Multiplatform (KMP)

When working with Kotlin Multiplatform:

1. Place shared business logic in `commonMain` source set; use `expect`/`actual` declarations only for platform-specific APIs (file I/O, crypto, platform settings) to maximize code reuse.
2. Configure KMP targets in `build.gradle.kts` with `kotlin { androidTarget(); iosX64(); iosArm64(); iosSimulatorArm64(); jvm() }` and define shared source sets with `sourceSets { commonMain.dependencies { } }`.
3. Use Ktor Client in `commonMain` for networking with engine-specific implementations (`OkHttp` for Android, `Darwin` for iOS, `CIO` for JVM); configure serialization with `kotlinx.serialization` content negotiation.
4. Persist structured data with SQLDelight by defining `.sq` files in `commonMain`; generate type-safe Kotlin APIs and provide platform-specific `SqlDriver` implementations via `expect`/`actual`.
5. Share UI across platforms with Compose Multiplatform; define `@Composable` functions in `commonMain` and use `expect`/`actual` composables only for platform-specific views like maps or camera previews.
6. Handle dependency injection with Koin Multiplatform; define modules in `commonMain` with `module { singleOf(::Repository) }` and initialize with `startKoin { modules(sharedModule, platformModule) }` per platform.
7. Define `expect` declarations as interfaces or abstract classes when the platform difference is complex; keep `actual` implementations minimal and test them with platform-specific test source sets.
8. Structure the project with a `:shared` module for KMP code and platform app modules (`:androidApp`, `:iosApp`); the iOS app consumes the shared module as a framework generated by the Kotlin/Native compiler.
9. Configure iOS framework export in Gradle with `iosTarget.binaries.framework { baseName = "Shared"; isStatic = true }` and integrate into Xcode via SPM package or CocoaPods plugin.
10. Use `kotlinx.coroutines` in `commonMain` for async code; on iOS, ensure the main dispatcher is available by adding `kotlinx-coroutines-core` and calling coroutines from Swift via `@MainActor` bridging.
11. Write shared tests in `commonTest` using `kotlin.test` (`@Test`, `assertEquals`, `assertFails`); run with `./gradlew allTests` to execute across all configured targets in one command.
12. Publish shared modules to Maven repositories with `maven-publish` plugin; configure artifact coordinates in `build.gradle.kts` and publish platform-specific artifacts automatically with `./gradlew publish`.
