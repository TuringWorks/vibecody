# Audio & Music Production

When working with audio and music production:

1. Select a DAW that fits the workflow: Pro Tools for professional recording studios and post-production (industry standard for mixing and film audio), Ableton Live for electronic music production and live performance (session view, warping, and real-time manipulation), Logic Pro for songwriting and composition on macOS (comprehensive built-in instruments and Apple ecosystem integration), and Reaper for budget-conscious production with deep customization.
2. Apply recording fundamentals by setting gain staging so input levels peak between -18 and -12 dBFS to leave headroom, placing microphones according to the source (large diaphragm condenser 6-12 inches for vocals, dynamic close-mic on guitar amps, stereo pair for room ambience), using pop filters for vocals, recording at 24-bit/48kHz minimum, and always capturing a few seconds of room tone.
3. Program MIDI effectively by quantizing to grid but humanizing with velocity variation and slight timing offsets, layering virtual instruments for richer textures, using automation lanes for expression and modulation, recording real-time knob movements for organic feel, and bouncing MIDI to audio once the performance is finalized to reduce CPU load.
4. Mix with a structured approach: start with static levels (faders only, no plugins), pan instruments to create stereo width (lead vocal and bass center, guitars and keys spread), apply subtractive EQ first to remove problem frequencies before boosting, use compression with intent (2-4 dB of gain reduction for gentle control, higher ratios for effect), and add reverb and delay on send/return buses rather than inserts for shared ambience.
5. Master to loudness standards by targeting -14 LUFS integrated for streaming platforms (Spotify, Apple Music), -16 LUFS for podcasts, and -24 LUFS for broadcast; use a limiter as the final plugin with no more than 2-3 dB of gain reduction, apply gentle master bus EQ and multiband compression, maintain a true peak ceiling of -1 dBTP, and always A/B compare with reference tracks.
6. Design sounds from scratch using synthesis (subtractive for classic analog tones, FM for bells and metallic textures, wavetable for evolving pads, granular for atmospheric textures), layer sampled recordings with synthesized elements, use foley recording for realistic sound effects (contact microphones for impacts, hydrophones for underwater), and process through creative effects chains (distortion, pitch shifting, time stretching).
7. Produce podcasts with a consistent workflow: record in a treated room or use dynamic microphones (SM7B, Rode PodMic) to reject room noise, edit with strip silence to remove dead air, apply a channel strip (high-pass filter at 80Hz, gentle compression, de-esser), match loudness between speakers, add intro/outro music at appropriate levels, and export as mono MP3 at 128kbps or stereo at 192kbps with proper ID3 tags.
8. Choose audio formats based on use case: WAV or AIFF (uncompressed, for production and archival), FLAC (lossless compression, 50-60% file size of WAV), MP3 at 320kbps (lossy, for general distribution), AAC at 256kbps (better quality than MP3 at same bitrate, Apple ecosystem), and Ogg Vorbis (open-source lossy, used in games); always archive masters as WAV 24-bit at the session sample rate.
9. Work with spatial audio by mixing in Dolby Atmos using a 7.1.4 bed with object-based audio for discrete elements, creating binaural mixes for headphone delivery, using Atmos-compatible DAWs (Pro Tools, Logic Pro, Nuendo), placing objects in 3D space with automation, and checking mixes in both spatial and stereo fold-down to ensure compatibility.
10. Organize sessions with a clear structure: use consistent track naming conventions (e.g., VOX_Lead, GTR_Rhythm_L, DRM_Kick), color-code by instrument group, create bus/group tracks for drums, vocals, instruments, and effects, use markers for song sections, save session templates for recurring project types, and maintain a separate folder structure for raw recordings, bounces, and exports.
11. Build plugin chains with signal flow in mind: insert plugins in logical order (corrective EQ, compression, tonal EQ, saturation, time-based effects), use pre-fader sends for effects that should remain constant regardless of fader position, post-fader sends for effects that track the dry level, keep parallel processing on separate buses, and bypass plugins regularly to confirm they improve the mix.
12. Restore and clean audio by using spectral editing (iZotope RX, Adobe Audition) to remove clicks, pops, and hums, applying adaptive noise reduction to capture and subtract noise profiles, using de-clip algorithms for distorted recordings, manually editing out mouth noises and breaths where appropriate, and always working on a copy of the original file to preserve the source material.
