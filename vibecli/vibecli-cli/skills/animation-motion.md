# Animation & Motion Design

When working with animation and motion design:

1. Apply the 12 principles of animation (Disney) as the foundation for all motion work: squash and stretch for weight and flexibility, anticipation to prepare the viewer, staging to direct attention, straight ahead versus pose to pose for different workflows, follow through and overlapping action for natural movement, slow in and slow out (easing) for realistic acceleration, arcs for organic motion paths, secondary action to enrich the scene, timing to control the feel of movement, exaggeration for clarity and appeal, solid drawing for dimensional form, and appeal for engaging character design.
2. Master keyframing and easing curves by placing keyframes only at meaningful positions (start, peak, end of a movement), using ease-in for objects coming to rest, ease-out for objects starting to move, ease-in-out for natural transitions between states, and customizing bezier curves in the graph editor rather than relying solely on presets; study timing charts to understand spacing between frames.
3. Write After Effects expressions to automate repetitive animation: use wiggle(frequency, amplitude) for organic randomness, loopOut("cycle") to repeat keyframe sequences, time*speed for continuous rotation, linear() and ease() for remapping values between ranges, and link properties across layers with pickwhip references; keep expressions concise and add comments for maintainability.
4. Implement CSS and JavaScript web animations by using CSS transitions for simple state changes (hover, focus), CSS @keyframes for looping or multi-step animations, GSAP (GreenSock) for complex sequenced timelines with ScrollTrigger for scroll-based animation, and Framer Motion for React component animations with declarative syntax (animate, variants, layout animations); prefer transform and opacity properties for GPU-accelerated performance.
5. Export and integrate Lottie animations for lightweight web and mobile delivery by designing in After Effects with supported features only (no expressions that Lottie cannot parse, no 3D layers, no track mattes beyond simple alpha), exporting with the Bodymovin plugin as JSON, testing in the LottieFiles preview tool, keeping file sizes under 50KB for UI animations, and implementing with lottie-web (web), lottie-ios, or lottie-android with proper playback controls.
6. Animate SVGs using CSS animations for simple transforms and opacity, SMIL for path morphing and complex attribute animation (though browser support is limited), JavaScript libraries (GSAP, anime.js, Snap.svg) for cross-browser compatibility, stroke-dasharray and stroke-dashoffset for line drawing effects, and viewBox manipulation for zoom and pan animations; inline SVGs for best animation control.
7. Set up character rigging by defining a skeletal hierarchy (root, spine, limbs, extremities), using inverse kinematics (IK) for natural limb movement, creating control handles for easy animator manipulation, building blend shapes or morph targets for facial expressions, testing the rig through full range of motion before animating, and using tools like Spine, DragonBones, or After Effects DUIK for 2D character rigging.
8. Storyboard motion design by sketching key frames that represent the most important moments, annotating timing and transitions between frames, indicating camera movement and scale changes, noting audio sync points (beats, dialogue cues), creating animatics (timed storyboard sequences) to test pacing before full production, and getting stakeholder approval on the animatic before proceeding to final animation.
9. Control timing and spacing precisely: use fewer frames between keyframes for fast, snappy motion and more frames for slow, graceful movement; typical UI animations should last 200-500ms, page transitions 300-700ms; avoid animations longer than 1 second for interface elements; stagger multiple elements by 50-100ms offsets for cascading effects; and always match animation duration to the emotional intent of the interaction.
10. Design UI micro-interactions for buttons (scale down on press, spring back on release), toggles (smooth slide with color transition), form validation (shake for errors, checkmark for success), loading states (skeleton screens, progress indicators with meaningful progress), pull-to-refresh (elastic overscroll with icon rotation), and state transitions (cross-fade, shared element transitions); keep micro-interactions subtle and consistent across the interface.
11. Build loading and transition animations that communicate system status: use skeleton screens for content loading, determinate progress bars when duration is known, indeterminate spinners or pulsing indicators when unknown, page transitions that maintain spatial context (slide in direction of navigation), and shared element transitions that morph from one state to another; ensure animations do not block user interaction and can be interrupted.
12. Optimize animation performance by using CSS transform and opacity (GPU-composited properties) instead of layout-triggering properties (width, height, top, left), batching DOM reads and writes, using requestAnimationFrame for JavaScript animations, will-change to hint browser compositing (use sparingly), reducing the number of animated elements on screen simultaneously, testing on low-end devices, and respecting prefers-reduced-motion media query for accessibility.
