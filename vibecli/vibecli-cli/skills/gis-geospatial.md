# GIS & Geospatial Analysis

When working with geographic information systems and spatial data:

1. Understand and manage GIS data types correctly — distinguish between vector data (points for discrete locations like wells or sensors, lines for linear features like roads or rivers, polygons for area features like parcels or watersheds) and raster data (continuous grid cells for elevation, temperature, land cover, satellite imagery), choose the format appropriate to the analysis (vector for discrete features and attribute queries, raster for continuous surface analysis and remote sensing), use standard formats (Shapefile, GeoPackage, GeoJSON for vector; GeoTIFF, Cloud Optimized GeoTIFF for raster), and understand topology rules (polygons must not overlap, lines must connect at nodes) to maintain data integrity.

2. Handle coordinate systems and projections with precision — always define the coordinate reference system (CRS) for every dataset, distinguish between geographic coordinate systems (latitude/longitude on an ellipsoid, e.g., WGS 84 EPSG:4326) and projected coordinate systems (flat plane in meters or feet, e.g., UTM zones, State Plane), choose projections that minimize distortion for the analysis purpose (equal-area for density maps, conformal for navigation, equidistant for distance measurements), reproject all layers to a common CRS before spatial analysis (use on-the-fly projection for display only), be aware of datum transformations when converting between NAD 27, NAD 83, and WGS 84, and store coordinates with sufficient precision (6 decimal places for degrees gives ~10 cm accuracy).

3. Perform spatial analysis using appropriate techniques — use buffer analysis to create proximity zones around features (fixed distance or variable based on attributes), overlay operations (intersect, union, identity, spatial join) to combine information from multiple layers, interpolation methods (IDW for local influence, Kriging for statistically optimal estimation, spline for smooth surfaces) to create continuous surfaces from point samples, network analysis for routing, service areas, and closest facility problems, viewshed and line-of-sight for terrain visibility, watershed delineation from DEMs for hydrological analysis, and spatial statistics (Moran's I for clustering, hot spot analysis, kernel density) to identify patterns. Always validate analysis parameters (buffer distance, cell size, search radius) against the data resolution and the question being answered.

4. Use ArcGIS Pro for enterprise GIS workflows — organize work in projects (.aprx) with maps and layouts, use the geoprocessing framework and ModelBuilder or ArcPy for repeatable analysis workflows, leverage ArcGIS Online and Portal for sharing web maps and apps with stakeholders, publish feature services for collaborative editing and data collection (Field Maps, Survey123), use ArcGIS Pro's built-in 3D analysis capabilities (scene layers, 3D analyst) for terrain and urban modeling, configure branch versioned editing for multi-user geodatabases, and automate recurring tasks with ArcPy scripts scheduled through Windows Task Scheduler or ArcGIS Notebooks.

5. Leverage QGIS as a powerful open-source GIS platform — use QGIS for projects where licensing cost is a concern or for maximum format compatibility, extend functionality with plugins from the QGIS Plugin Repository (QuickMapServices for basemaps, qgis2web for web map export, Semi-Automatic Classification Plugin for remote sensing), use the Processing Toolbox which integrates GRASS GIS, SAGA, and GDAL/OGR algorithms, automate workflows with PyQGIS scripting and the graphical modeler, create print layouts with map atlases for batch map production, connect to PostgreSQL/PostGIS databases for enterprise data management, and contribute to the open-source community by reporting bugs and sharing plugins.

6. Process and analyze remote sensing and satellite imagery — obtain imagery from appropriate sources (Landsat for long-term monitoring at 30m, Sentinel-2 for 10m multispectral analysis, commercial providers like Planet or Maxar for sub-meter resolution), apply atmospheric correction to convert raw digital numbers to surface reflectance, calculate spectral indices (NDVI for vegetation health, NDWI for water bodies, NDBI for built-up areas, NBR for burn severity), perform supervised classification (Random Forest, Support Vector Machine, Maximum Likelihood) with representative training samples and accuracy assessment, use change detection techniques (image differencing, post-classification comparison) to monitor land use change over time, and leverage cloud platforms (Google Earth Engine, Microsoft Planetary Computer) for large-scale imagery processing without local storage constraints.

7. Design geodatabases for efficient data management — define a logical data model before creating the database (entities, attributes, relationships, domains), use coded value and range domains to enforce data quality (e.g., road type must be one of: highway, arterial, collector, local), implement subtypes to categorize features with different default values and validation rules, create relationship classes to link related tables (parcels to owners, pipes to inspections), use topology rules to enforce spatial integrity (parcels must not overlap, water lines must connect at junctions), design feature datasets with a common spatial reference for topologically related layers, and document the data model with a data dictionary describing each field's purpose, type, and valid values.

8. Build web maps and applications for data sharing — choose the appropriate web mapping technology: Leaflet.js for lightweight, mobile-friendly maps; Mapbox GL JS for high-performance vector tile rendering and custom styling; OpenLayers for advanced OGC standards support; ArcGIS JavaScript API for integration with ArcGIS services; use vector tiles for fast rendering of large datasets, configure pop-ups and labels for user interaction, implement layer controls and legends for multi-layer maps, optimize performance with spatial indexing, level-of-detail rendering, and tile caching, and ensure accessibility (keyboard navigation, screen reader support, color-blind-friendly palettes).

9. Implement geocoding and routing services — use geocoding services (Google, Esri, OpenCage, Nominatim) to convert addresses to coordinates, batch geocode large datasets with error handling and match score thresholds (reject matches below 80% confidence), implement reverse geocoding for converting GPS coordinates to addresses, configure routing with appropriate network datasets (OpenStreetMap for open data, HERE or TomTom for commercial accuracy), account for turn restrictions, one-way streets, and road hierarchy in network models, calculate service areas (isochrones) for accessibility analysis, and implement multi-stop route optimization for logistics applications.

10. Work with spatial SQL and PostGIS for database-driven analysis — use PostGIS to extend PostgreSQL with spatial data types and functions, store geometry with appropriate SRID using the `geometry` or `geography` column types, create spatial indexes (`CREATE INDEX ... USING GIST`) for query performance, use spatial predicates (`ST_Intersects`, `ST_Within`, `ST_DWithin`) for filtering by location, perform spatial operations (`ST_Buffer`, `ST_Union`, `ST_Intersection`) directly in SQL, calculate measurements (`ST_Area`, `ST_Length`, `ST_Distance`) with proper CRS considerations, use `ST_Transform` to reproject geometries on the fly, and leverage window functions with spatial ordering for nearest-neighbor queries and spatial aggregation.

11. Process LiDAR point cloud data for terrain and feature extraction — manage large point cloud datasets in LAS/LAZ format using tools like LAStools, PDAL, or CloudCompare, classify points into ground, vegetation, buildings, and noise using automated algorithms with manual quality checks, generate Digital Elevation Models (DEM) and Digital Surface Models (DSM) from classified ground and first-return points respectively, extract building footprints and tree canopy from above-ground point classifications, calculate Canopy Height Models (CHM = DSM - DEM) for forestry applications, use intensity and return number attributes for land cover classification, and optimize processing by tiling large datasets and working with spatial subsets.

12. Apply cartographic principles for effective map communication — choose a visual hierarchy that emphasizes the map's primary message (the most important layer should be most visually prominent), use appropriate color schemes from ColorBrewer (sequential for ordered data, diverging for data with a meaningful midpoint, qualitative for categorical data), limit the number of classes in thematic maps (5-7 for choropleth maps using natural breaks, quantiles, or equal intervals as appropriate), include essential map elements (title, legend, scale bar, north arrow, source citation, date), use appropriate labeling (Maplex for ArcGIS, rule-based labeling in QGIS) with conflict resolution, design for the output medium (screen resolution for web, 300 DPI for print), and create map series or atlases for systematic coverage of large areas.
