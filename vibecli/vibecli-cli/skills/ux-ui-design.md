# UX/UI Design

When designing user experiences and interfaces:

1. Conduct user research early using interviews, surveys, and usability tests to ground decisions in real user needs rather than assumptions.
2. Start with low-fidelity wireframes and information architecture before investing in high-fidelity mockups; validate structure before polish.
3. Build on a design system with reusable component libraries to ensure consistency, speed up development, and reduce design drift.
4. Use Figma auto-layout and variants to create responsive, maintainable component designs that translate cleanly to code.
5. Design for accessibility from the start targeting WCAG 2.1 AA: sufficient contrast, keyboard navigation, screen reader support, and focus management.
6. Apply responsive design patterns (fluid grids, container queries, breakpoints) and adopt a mobile-first approach to handle the smallest viewport first.
7. Structure information architecture with clear navigation hierarchies, consistent labeling, and logical content grouping so users find what they need quickly.
8. Design micro-interactions and transitions that provide feedback, guide attention, and communicate state changes without distracting from the task.
9. Apply color theory (contrast, harmony, semantic meaning) and typography best practices (hierarchy, readability, limited font families) to reinforce usability.
10. Prepare thorough design handoffs with specs, design tokens, exportable assets, and annotated flows so engineers can implement without guesswork.
11. Run iterative user testing sessions (moderated and unmoderated) after each major design milestone and feed findings back into the next iteration.
12. Adopt a mobile-first design approach where constraints of smaller screens force prioritization of content and interactions before scaling up to larger viewports.
