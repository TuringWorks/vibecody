# Agent Thought Stream Visibility

When exposing agent reasoning and chain-of-thought to users or downstream systems:

1. **Extracting Thinking Blocks** — Models that support explicit thinking blocks (e.g., Claude extended thinking) emit structured `<thinking>` spans separate from the final response. Extract these spans verbatim before stripping them from the user-facing output. For models without native thinking blocks, extract sentences beginning with hedging language ("I need to...", "Let me check...", "First, I should...") as implicit thought fragments.
2. **Thought Categorization Heuristics** — Classify each thought fragment into one of five categories: `planning` (deciding next action), `reasoning` (analyzing information), `uncertainty` (expressing doubt or alternatives), `self-correction` (revising a prior step), and `tool-prep` (constructing a tool call). Assign category via keyword matching and context window; surface categories in the thought stream UI to help users quickly navigate the reasoning trace.
3. **Confidence Scoring** — Assign a confidence score (0.0–1.0) to each thought fragment based on: linguistic certainty markers (high confidence: "I know", "clearly"; low confidence: "I think", "maybe", "not sure"), presence of contradiction with previous thoughts, and whether the thought was later revised by a self-correction fragment. Display low-confidence thoughts (<0.4) with a visual indicator in the UI.
4. **When High-Uncertainty Thoughts Warrant Human Review** — Automatically flag thought sequences for human review when: three or more consecutive uncertainty-category thoughts appear, a self-correction fragment reverses a prior confirmed decision, confidence score drops below 0.3 for a thought directly preceding a destructive action (file delete, deploy, API call with side effects), or the model expresses ambiguity about which of two conflicting instructions to follow.
5. **Exporting Thoughts for Debugging** — Export the full thought stream as a JSONL file where each line is `{"timestamp": "...", "turn": N, "category": "...", "confidence": N, "text": "..."}`. Include the export path in the task completion report. For CI/automated contexts, emit thought stream to stderr so it is captured in build logs without polluting stdout. Never suppress thought export when a task fails — the reasoning trace is the primary debugging artifact.
6. **Streaming to the UI** — Stream thought fragments in real time as they are generated; do not buffer them until the full response is complete. Use a distinct visual channel (collapsible sidebar, dimmed text, or progress feed) to separate thoughts from final output. Allow users to collapse the thought stream by default but expand on demand. Thought stream latency must not delay the final response delivery.
7. **Privacy and Redaction** — Scan thought stream text for PII (email addresses, phone numbers, social security numbers, API keys) before displaying or logging. Redact detected PII with `[REDACTED]` placeholders. Thoughts referencing internal system prompts or confidential instructions must be fully suppressed from user-visible output. Apply the same redaction rules to exported thought stream files.
8. **Thought Stream Retention Policy** — Retain thought streams for the same duration as the associated task logs (default: 30 days). Thought streams from tasks involving sensitive data (flagged by policy) must be encrypted at rest using the workspace key. Provide a user-accessible purge command to delete thought streams for a specific session without deleting the task results.
